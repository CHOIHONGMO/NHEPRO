# NHEPRO 프로젝트 배포 방법 가이드 (프로젝트_배포방법.md)

NHEPRO 프로젝트의 소스 코드를 빌드하고 서버(WAS)에 배포하는 표준 프로세스를 설명합니다. 본 프로젝트는 Maven 빌드 및 WAS(Apache Tomcat) 기반으로 동작합니다.

---

## 1. 배포 전제 조건 및 아키텍처

* **배포 산출물**: 
  * `NHeProFront-1.0-SNAPSHOT.war` (사용자 포털)
  * `NHeProOperator-1.0-SNAPSHOT.war` (운영자 포털)
* **권장 WAS 버전**: **Apache Tomcat 9.0 이하** (예: 내장 디렉토리 내 `Tomcat 9.0.118`)
  > [!IMPORTANT]
  > 본 프로젝트는 Spring 3.2.18 및 `javax.servlet` 네임스페이스 기반의 웹 애플리케이션입니다. 
  > Tomcat 10.0 이상부터는 `jakarta.servlet` 네임스페이스를 사용하므로, 라이브러리 충돌 및 실행 에러가 발생합니다. 반드시 **Tomcat 9.0 이하** 버전에 배포하셔야 합니다.

---

## 2. 1단계: Maven 프로파일(Profile) 선택

각 웹 프로젝트는 배포 대상 환경에 따라 설정 정보(Conf, JS 라이선스 키, DB 접속 정보 등)를 다르게 포함하도록 Maven Profile이 구성되어 있습니다.

| 프로파일 ID (`-P`) | 대상 환경 | 주요 특징 |
| :--- | :--- | :--- |
| **local** (기본값) | 개발자 로컬 PC | 로컬 DB 연동 및 개발용 설정 파일 포함 |
| **development** | 개발/스테이징 서버 | 개발 환경용(`profiles/development/conf`) 설정 파일 적용 |
| **production** | 실운영 서버 | 운영 서버용(`profiles/production/conf`) 설정 파일 및 MagicLine, RealGrid 실운영 라이선스 반영 |

---

## 3. 2단계: 프로젝트 빌드 및 WAR 패키징

빌드는 터미널에서 내장 Maven을 통해 진행합니다. (VS Code 터미널 이용 권장)

### 3.1. NHeCommon 사전 설치 (필수)
웹 프로젝트가 의존하는 공통 모듈을 먼저 빌드하여 로컬 저장소에 반영해야 합니다.
```powershell
$env:JAVA_HOME='C:\ST-onesIDE\java\java-1.8'
& 'C:\ST-onesIDE\server\apache-maven-3.9.16\bin\mvn.cmd' clean install -f NHeCommon/pom.xml
```

### 3.2. 웹 애플리케이션(Front / Operator) 빌드
원하는 프로파일 인자(`-P`)를 지정하여 패키징 명령어를 실행합니다.

#### 개발(Development) 서버 배포용 빌드 예시
```powershell
# NHeProFront 빌드
& 'C:\ST-onesIDE\server\apache-maven-3.9.16\bin\mvn.cmd' clean package -f NHeProFront/pom.xml -Pdevelopment

# NHeProOperator 빌드
& 'C:\ST-onesIDE\server\apache-maven-3.9.16\bin\mvn.cmd' clean package -f NHeProOperator/pom.xml -Pdevelopment
```

* **빌드 산출물 경로**:
  * `NHeProFront/target/NHeProFront-1.0-SNAPSHOT.war`
  * `NHeProOperator/target/NHeProOperator-1.0-SNAPSHOT.war`

## 4. 3단계: WAS (Apache Tomcat 9.0) 배포 절차

패키징된 WAR 파일을 Tomcat 서버에 배포하는 방식은 크게 두 가지가 있습니다.

### 방법 A: webapps 폴더에 직접 복사 (운영 환경 권장)
1. Tomcat 서버를 중지합니다.
2. 빌드된 `war` 파일의 이름을 각각 서비스 도메인 경로에 맞게 변경합니다. (예: `ROOT.war` 혹은 `front.war`, `operator.war`)
3. 변경한 `war` 파일을 Tomcat 홈 디렉토리 하위의 `webapps/` 폴더에 복사합니다.
4. Tomcat 서버를 시작하면 자동으로 WAR 압축이 해제되며 배포가 완료됩니다.
   * Tomcat 구동 스크립트: `C:\ST-onesIDE\server\apache-tomcat-9.0.118\bin\startup.bat`

### 방법 B: Tomcat server.xml 설정 변경 (개발 환경 권장)
WAR 파일을 매번 복사하지 않고, 빌드된 디렉토리를 Tomcat 컨텍스트로 직접 지정하여 배포합니다.
1. `C:\ST-onesIDE\server\apache-tomcat-9.0.118\conf\server.xml` 파일을 엽니다.
2. `<Host>` 태그 내에 아래와 같이 `<Context>` 설정을 추가합니다. (경로는 빌드 타겟에 맞춤)

```xml
<Host name="localhost" appBase="webapps" unpackWARs="true" autoDeploy="true">
    
    <!-- 사용자 포털 (Front) 설정 -->
    <Context path="/front" docBase="C:/ST-onesIDE/workspace/NHEPRO/NHeProFront/target/NHeProFront-1.0-SNAPSHOT" reloadable="true" />
    
    <!-- 운영자 포털 (Operator) 설정 -->
    <Context path="/operator" docBase="C:/ST-onesIDE/workspace/NHEPRO/NHeProOperator/target/NHeProOperator-1.0-SNAPSHOT" reloadable="true" />

</Host>
```
3. 설정 후 Tomcat을 실행하면, 매번 WAR를 복사할 필요 없이 `mvn package` 명령어 실행 후 즉시 반영됩니다.
