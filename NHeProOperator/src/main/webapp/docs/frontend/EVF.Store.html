Component: Store

EVF.Store

Store 객체는 프론트엔드와 서버와의 AJAX 통신 메소드를 정의한 유틸리티입니다.
Since:
  • 1.0.0
Example
<script type="text/javascript">
  function doSearch() {
    var store = new EVF.Store();
    store.setParameter('KEY', 'VALUE');
    store.load('${SERVER_URL}', function() { // callback function ... });
  }
</script>

doFileUpload(callbackFunction)

이 함수를 호출하면 화면의 모든 파일매니저의 업로드를 시작시키고, 업로드가 모두 종료되면 callbackFunction을 실행합니다.
파일매니저 관련 URL은 load() 메소드와는 별개로 EverUX 프레임워크 전용으로 지정되어 있어서 따로 지정할 필요가 없습니다.
파일매니저가 처음 페이지에 로드될 때 자기 자신에게 파일 UUID가 부여되지 않았다면 자동으로 생성하여 자신에게 지정하므로 이 메소드를 호출했을 때 자기 자신의 UUID 대로 서버에 업로드된 후에 콜백함수를 호출하게 됩니다. 파일업로드와 폼 데이터 조작은 동시에 처리할 수 없기 때문에 아래의 예제 코드처럼 별도로 구현해야 합니다.
Parameters:
Name Type Description
callbackFunction function 콜백함수
Example
var store = new Store();
store.doFileUpload(
    store.load('URL', function() {
        // do something
    });
);

getGridData(gridObject, dataFlag)

서버에 전송할 데이터의 유형을 설정합니다.
dataFlag를 all로 설정할 경우 그리드의 전체 데이터를, sel로 설정할 경우 선택된 데이터만 넘기도록 Store 객체에 설정합니다.
Parameters:
Name Type Description
gridObject Object 그리드 객체
dataFlag string all|sel
Example
var grid = EVF.getComponent(GRID_ID);
var store = new Store();
store.getGridData(grid, 'sel');  // 체크된 그리드 데이터만 서버에 전송하겠다는 설정
store.load('URL', function() {
    ...
});

getResponseCode() → {string}

서버에서 넘어온 통신 결과코드를 리턴합니다.
서버에서 EverHttpResponse.setResponseCode() 로 코드를 설정했을 때만 반환됩니다. 필요한 코드가 있다면 설정한 후 개발합니다.
Returns:
서버에서 전송된 코드값
Type
string
Example
var store = new Store();
store.load(URL, function() {
    this.getResponseCode();   // 콜백함수 내의 this 객체는 store 객체를 가리킵니다.
});

getResponseMessage() → {string}

서버에서 넘겨받은 메시지를 리턴합니다.
서버에서 EverHttpResponse.setResponseMessage() 로 메시지를 설정했을 때만 반환됩니다. ajax 통신 후 메시지를 출력시킬 수 있습니다.
Returns:
서버메시지
Type
string
Example
var store = new Store();
store.load(URL, function() {
    this.getResponseMessage();   // 콜백함수 내의 this 객체는 store 객체를 가리킵니다.
});

load(url, afterloadFunction, useMask)

서버와의 AJAX 통신을 시작합니다. 페이지 내의 모든 입력컴포넌트의 값을 읽어들여 서버로 전송합니다.
입력컴포넌트를 사용하지 않은 form 객체들의 값은 서버로 전송되지 않으니 store.setParameter(KEY, VALUE); 를 사용하여 따로 지정해야 전송됩니다.
Parameters:
Name Type Description
url string 서버 URL
afterloadFunction function 통신 후 실행할 콜백함수
useMask boolean Mask 사용여부

setGrid(gridParam)

그리드 객체가 담긴 배열을 Store 객체에 설정합니다.
Store 객체에서 load() 함수를 실행시키면 콜백함수가 실행되고 서버에서 받아온 객체 중 setGrid()로 지정한 그리드와 매치되는 그리드 객체가 있으면 자동으로 그 그리드 패널에 해당 그리드 데이터를 자동으로 삽입하여 재조회해줍니다. 쉽게 말해 이 코드를 설정 안해주면 그리드를 조회해도 데이터가 그리드에 보이지 않습니다.
웹페이지에 그리드를 포함하고, 그리드 데이터를 조회하는 함수를 개발 시 반드시 설정해야하는 코드입니다.
Parameters:
Name Type Description
gridParam Array.<EVF.GridPanel> 그리드 객체 배열
Example
var grid = EVF.C('grid');
     var store = new Store();
     store.setGrid([grid]);
     store.load('URL', function() { ... });

validate(containerId) → {boolean}

페이지 전체의 입력컴포넌트 혹은 파라미터로 지정된 컨테이너 내의 입력컴포넌트들의 데이터 유효성을 체크합니다. (체크내용: 필수입력 여부)
Parameters:
Name Type Description
