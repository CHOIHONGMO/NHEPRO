How-To develop with EverUX Framework.

이 페이지는 프레임웍을 이용해 간단한 페이지 하나를 개발해볼 수 있도록 안내하기 위해 작성된 문서입니다.

Controller Class 개발

클래스를 작성합니다.

@Controller
@RequestMapping(value = "/sample")
public class TestController extends BaseController {
  // ...
}

개발하는 JSP를 맵핑시킬 메소드를 작성합니다. 모든 JSP는 보안 상 WEB-INF/view 디렉토리에 위치시켜야 하며, 웹상에서는 이 위치를 직접 액세스할 수 없기 때문에 반드시 이 맵핑과정을 거쳐야 합니다.

@Controller
@RequestMapping(value = "/sample")
public class TestController extends BaseController {
  @RequestMapping(value = "/testPage/view")
    public String view(EverHttpRequest req, EverHttpResponse resp) throws Exception {
      return "testPage";    // JSP 파일명 (경로가 있다면 경로까지 지정)
  }
}

결과적으로 클래스의 RequestMapping과 합쳐져서(/sample + /view) /sample/view 라는 맵핑이 완성되었고, EverUX에서는 *.so 확장자를 반드시 붙여야 해당 맵핑을 찾아서 컨트롤러의 해당 메소드를 호출하게 됩니다. (http://localhost/sample/view.so)

JSP 개발

이제 맵핑을 마쳤으니 JSP를 개발해 봅니다.

<%@ page import="net.htmlparser.jericho.Source" %>
<%@ page import="net.htmlparser.jericho.OutputDocument" %>
<%@ page language="java" contentType="text/html; charset=UTF-8" pageEncoding="UTF-8"%>
<%@taglib prefix="e" uri="http://www.st-ones.com/eversrm" %>
<e:ui>
    <script type="text/javascript">
        ...
    </script>
    <e:window id="화면ID" onReady="init" name="test" title="페이지명">
        ...
    <e:window>
</e:ui>

위의 코드는 모든 JSP의 공통된 구조입니다. 화면을 개발하려면 e:window 태그 밑에 작성합니다. 아래의 코드는 간단한 조회화면을 코딩한 화면입니다. 각 컴포넌트의 이름들이 생소할 수 있지만 주석에 씌여진 것과 같이 이 태그들은 모두 해당 html 태그로 결국 변환이 되어 브라우저에서 표시되게 됩니다.

아래의 코드를 작성한 후 Ever 솔루션의 화면 속성 관리에서 폼, 그리드 컬럼 정보를 먼저 선입력하고 난 후 해당 화면ID와 아래 코드에서 e:window 의 id 속성과 그 값을 같도록 하면 컴포넌트의 각 속성들에 입력한 특정 조건에 따라 해당 입력정보를 치환하여 보여줍니다.

<e:ui>
    <script type="text/javascript">
    ...
    </script>
    <e:window id="test" onReady="init" name="test" title="테스트 페이지">
        <e:searchPanel id="form" name="form" labelWidth="120" columnCount="2">    <!-- 테이블과 동일 -->
            <e:row>                                                               <!-- <tr>과 동일 -->
                <e:label for="input1" title="입력1" />                             <!-- <td>와 동일 -->
                <e:field>
                    <inputText id="input1" name="input1" ... />
                </e:field>
                <e:label for="input2" title="입력2" />
                <e:field>
                    <inputText id="input2" name="input1" ... />
                </e:field>
            </e:row>
        </e:searchPanel>
        <e:buttonBar id="buttonBar">
        <e:button id="button" label="조회" onClick="doSearch" />
        </e:buttonBar>
        <e:gridPanel id="grid" name="test" height="fit" />
    <e:window>
</e:ui>

Service Class 개발

화면에 조회 버튼이 있습니다. 이 버튼에 대한 서비스 클래스를 개발해 보겠습니다. 먼저 JSP에서 자바스크립트 함수를 추가합니다.

<e:ui>
    <script type="text/javascript">
        function doSearch() {
            var store = new EVF.Store();            // 통신을 하기 위해 store 객체를 생성합니다.
            var grid = EVF.getComponent('grid');    // 그리드 객체를 변수에 담습니다.
            store.getGridData(grid, 'all');         // 모든 그리드 데이터를 넘기려고 합니다.
            store.setParameter('mode', 'test');     // 특정 파라미터를 넘기려고 합니다.
            store.load('/sample/testPage/doSearch.so', function() {
                // 위의 URL로 ajax 통신을 요청하고, 결과메시지를 알림창으로 띄웁니다.
                alert(this.getResponseMessage());
            });
        }
    </script>
    ...

Controller 클래스에서 doSearch에 대한 기능을 호출해주기 위해 맵핑할 메소드를 하나 더 추가합니다.

@Controller
@RequestMapping(value = "/sample")
public class TestController extends BaseController {

    @Autowired
    TestService testService;

    @RequestMapping(value = "/view")
    public String view(EverHttpRequest req, EverHttpResponse resp) throws Exception {
        return "testPage";
    }

    @RequestMapper(value = "/doSearch")
    public void doSearch(EverHttpRequest req, EverHttpResponse resp) throws Exception {
        Map<String, String> formData = req.getFormData();
        Map<String, Object> gridData = req.getGridData("test");

        List<Map<String, Object>> resultGridData = testService.doSearch(formData);
        resp.setGridObject('grid', resultGridData);
        resp.setResponseCode('0001');
        resp.setResponseMessage('성공적으로 조회했습니다.');
    }
}

Service 클래스를 개발합니다.

    @Service(value = "testService")
    public class TestService extends BaseService {

    @Autowired
    public TestMapper testMapper;

    public List<Map<String, Object>> doSearch(Map<String, String> param) throws Exception {
        return testMapper.doSearch(param);
  }
}

package test;

public interface TestMapper {
    public List<Map<String, Object>> doSearch(Map<String, String> param);
}

여기까지 간단하게 샘플페이지를 작성하는 방법을 작성하였습니다.

보다 자세하고 복잡한 화면의 형태는 이 페이지에서 모두 설명할 수 없고, 기 개발된 화면의 소스를 참고하는 것이 더 빠를 것입니다.

이 페이지의 최상단 메뉴의 Classes 는 EverUX Front-End에서 각 컴포넌트 별로 스크립트로 제어할 수 있는 API가 정리되어 있습니다. Abstract가 prefix로 붙은 클래스들은 모두 super 클래스로써 단독으로 사용되지 않고 컴포넌트들이 상속받아서 구현되는 클래스입니다.
