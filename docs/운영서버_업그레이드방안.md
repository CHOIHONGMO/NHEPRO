# JEUS 7 ➔ JEUS 8 운영 서버 업그레이드 방안

본 문서는 운영 서버의 WAS를 JEUS 7에서 JEUS 8 버전으로 업그레이드하는 과정에서 고려해야 할 주요 기술적 사항과 NHEPRO 프로젝트 내에서 변경이 필요한 사항을 설명합니다.

---

## 1. 프로젝트 내 주요 소스코드 변경 사항

### ① `jeus-web-dd.xml` XML 스키마 버전 상향
* **대상 파일:** [jeus-web-dd.xml](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProFront/src/main/webapp/WEB-INF/jeus-web-dd.xml#L2)
* **변경 내용:**
  JEUS 8의 웹 컨테이너 스펙과 설정 규칙을 호환하기 위해 `version` 속성을 `7.0`에서 `8.0`으로 상향 조정합니다.
  ```xml
  <!-- AS-IS -->
  <jeus-web-dd xmlns="http://www.tmaxsoft.com/xml/ns/jeus" version="7.0">

  <!-- TO-BE -->
  <jeus-web-dd xmlns="http://www.tmaxsoft.com/xml/ns/jeus" version="8.0">
  ```

### ② 서버 로그 조회 경로 설정 외부화 (하드코딩 제거)
* **대상 파일:**
  * [WatcherController.java (Front)](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProFront/src/main/java/com/st_ones/common/eversrm/system/WatcherController.java)
  * [WatcherController.java (Operator)](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProOperator/src/main/java/com/st_ones/common/eversrm/system/WatcherController.java)
  * [nhepro.properties (Front)](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProFront/src/main/resources/nhepro.properties)
  * [nhepro.properties (Operator)](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProOperator/src/main/resources/nhepro.properties)
* **변경 내용:**
  기존 소스코드 내에 하드코딩되어 있던 JEUS 로그 디렉터리 경로(`/logs/tmax/jeus7/...`)를 `nhepro.properties`로 이관하고, `PropertiesManager.getString()`을 통해 가져오도록 수정합니다.
  * JEUS 8 업그레이드 배포 시, `nhepro.properties` 내 경로를 새 WAS 경로(예: `/logs/tmax/jeus8/...`)로 수정하기만 하면 추가 소스 컴파일 없이 대응이 가능합니다.

## 2. WAS 업그레이드 시 기술적 고려사항

### ① JDK 1.8 기반 런타임 호환성 검증
* **현황:** 프로젝트 컴파일 스펙은 JDK 1.8로 설정되어 있습니다 (`pom.xml` 소스/타깃 설정 완료).
* **고려사항:** JEUS 8 구동 시 JDK 1.8 환경을 표준으로 사용하므로 컴파일 오류 가능성은 희박하지만, 운영 서버 JVM 버전과의 일치 여부 및 메모리 설정 옵션(Garbage Collector 등)을 점검해야 합니다.

### ② Java EE 7 스펙 표준 준수 및 검증
* JEUS 8은 Java EE 7 표준(Servlet 3.1, JSP 2.3)을 지원합니다.
* Servlet 3.1 스펙 상승으로 인한 Non-blocking I/O 처리 변화, 세션 쿠키 설정 강화, 필터/리스너 호출 방식 등 표준 API 레벨에서의 엄격한 예외 처리가 발생할 수 있으므로 개발 및 스테이징 환경에서 주요 모듈(인증, 파일 다운로드 등)에 대한 집중 검증이 필요합니다.

### ③ 보안 솔루션 및 Native 라이브러리(JNI) 검증
* 프로젝트 내에서 시스템 경로로 직접 로드 중인 외부 암호화/인증 JNI 모듈(`MagicJCrypto`, `fasoo-jni` 등)이 JDK 1.8 및 JEUS 8 엔진 환경에서 충돌 없이 로드되는지 연동 테스트를 수행해야 합니다.

## 3. 운영 이관 가이드

### ① `domain.xml` 설정 신규 작성
* JEUS 7의 `domain.xml` 파일을 JEUS 8 엔진에 그대로 복사하면 스키마 버전 오류로 기동되지 않습니다.
* JEUS 8의 신규 도메인을 생성한 후 WebAdmin 또는 `jeusadmin` 콘솔을 이용해 다음 설정을 수동으로 다시 이관하여 새로 저장해야 합니다:
  * **데이터소스 (DataSource)** 연결 설정
  * **스레드 풀 (Thread Pool)** 옵션
  * **JVM 옵션 (Memory, System Properties 등)**
  * **세션 클러스터링 (Session Clustering)** 구성
