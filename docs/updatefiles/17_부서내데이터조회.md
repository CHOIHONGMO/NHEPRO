# 부서내 데이터 조회 권한 제어 방안 (RQ-PR-IT20260017)

본 문서는 로그인한 사용자의 조직 레벨(본부-부-단-국-팀)에 맞추어 하위 조직의 모든 구매요청 데이터를 조회할 수 있도록 제한하는 두 가지 해결책을 정의합니다.

---

## 해결책 1. 데이터베이스 계층형 쿼리(CONNECT BY / CTE) 기반 처리

로그인한 사용자의 부서코드(`#{ses.deptCd}`)를 기준으로 데이터베이스 단에서 실시간으로 계층 구조(하위 부서들)를 탐색하여 구매요청 데이터를 필터링하는 방식입니다.

### 1) MyBatis Mapper XML 쿼리 적용 방안
각 업무화면 조회 Mapper XML(예: [CPRI0010_Mapper.xml](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProFront/src/main/resources/mappers/com/st_ones/nhepro/CPRI/CPRI0010_Mapper.xml))의 조회 조건절에 데이터베이스별 계층 쿼리 분기 처리를 추가합니다.

```xml
AND EXISTS (
    SELECT 1 
      FROM STOCUSER UR
     WHERE UR.GATE_CD  = PRHD.GATE_CD
       AND UR.USER_ID  = PRHD.REQ_USER_ID -- 구매요청자 ID 매핑
       AND UR.DEL_FLAG = '0'
       AND UR.DEPT_CD IN (
           <if test='_databaseId == "oracle"'>
               SELECT DEPT_CD 
                 FROM STOCOGDP
                WHERE GATE_CD  = PRHD.GATE_CD
                  AND BUYER_CD = PRHD.BUYER_CD
                  AND DEL_FLAG = '0'
                START WITH DEPT_CD = #{ses.deptCd} -- 로그인 사용자 부서코드
              CONNECT BY PARENT_DEPT_CD = PRIOR DEPT_CD
           </if>
           <if test='_databaseId == "mssql"'>
               WITH DEPT_CTE AS (
                   SELECT DEPT_CD 
                     FROM STOCOGDP
                    WHERE GATE_CD  = PRHD.GATE_CD
                      AND BUYER_CD = PRHD.BUYER_CD
                      AND DEPT_CD  = #{ses.deptCd}
                      AND DEL_FLAG = '0'
                    UNION ALL
                   SELECT Child.DEPT_CD
                     FROM STOCOGDP Child
                     INNER JOIN DEPT_CTE Parent ON Child.PARENT_DEPT_CD = Parent.DEPT_CD
                    WHERE Child.GATE_CD  = PRHD.GATE_CD
                      AND Child.BUYER_CD = PRHD.BUYER_CD
                      AND Child.DEL_FLAG = '0'
               )
               SELECT DEPT_CD FROM DEPT_CTE
           </if>
       )
)
```

- **장점:** Java 소스코드 및 세션 클래스 변경 최소화.
- **단점:** 대용량 데이터 조회 시 실시간 CONNECT BY 쿼리로 인한 성능 부하 가능성 존재.

## 해결책 2. Java 세션 정보 캐싱 및 MyBatis `<foreach>` 활용 처리 (권장)

부서 권한 제어는 소속 회사별 설정에 따라 활성화 여부가 결정됩니다. 회사 정보 테이블(`STOCCUST`)에 부서내검색여부(`DEPT_SEARCH_FLAG`) 속성을 생성하고, 로그인 세션 생성 시 이 값이 `'1'`인 경우에만 하위 부서 목록(`subDeptList`)을 재귀적으로 조회하여 세션에 캐싱한 후 쿼리에 바인딩합니다.

### 1) DB 스키마 추가 (회사 정보 설정)
- **대상 테이블:** `STOCCUST` (고객사 테이블)
- **추가 컬럼:** `DEPT_SEARCH_FLAG` (`CHAR(1)` / Default `'0'`)
  - `0`: 부서내 검색 기능 비활성화 (권한 제한 없음)
  - `1`: 부서내 검색 기능 활성화 (자신의 하위 조직 데이터만 조회 제한)

### 2) [UserInfo.java](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeCommon/src/main/java/com/st_ones/common/login/domain/UserInfo.java) 세션 도메인 수정
로그인 사용자 세션 정보를 유지하는 객체에 회사별 부서내검색여부(`deptSearchFlag`)와 하위 부서 목록(`subDeptList`)을 저장할 수 있는 필드를 추가합니다.

```java
public class UserInfo extends BaseInfo {
    // ... 기존 필드 생략 ...
    private String deptSearchFlag;
    private java.util.List<String> subDeptList;

    public String getDeptSearchFlag() { return deptSearchFlag; }
    public void setDeptSearchFlag(String deptSearchFlag) { this.deptSearchFlag = deptSearchFlag; }

    public java.util.List<String> getSubDeptList() { return subDeptList; }
    public void setSubDeptList(java.util.List<String> subDeptList) { this.subDeptList = subDeptList; }
}
```

### 3) 로그인 프로세스 세션 정보 연동 ([LoginController.java](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeCommon/src/main/java/com/st_ones/common/login/web/LoginController.java))
세션 생성 메소드 `createLoginUserSession` 실행 시, 로그인 사용자의 소속 회사 설정(`deptSearchFlag`)이 `'1'`인 경우 하위 부서 목록을 조회하여 세션에 바인딩합니다.

```java
private void createLoginUserSession(EverHttpRequest req, EverHttpResponse resp, Map<String, UserInfo> resultInfo) throws Exception {

    UserInfo userInfo = resultInfo.get("baseInfo");
    if (userInfo != null) {
        // 소속 회사의 부서내검색여부(deptSearchFlag)가 '1'인 경우에만 처리
        if ("1".equals(userInfo.getDeptSearchFlag())) {
            java.util.Map<String, String> param = new java.util.HashMap<String, String>();
            param.put("GATE_CD", userInfo.getGateCd());
            param.put("BUYER_CD", userInfo.getCompanyCd());
            param.put("DEPT_CD", userInfo.getDeptCd());
            
            // 로그인 서비스에서 재귀적 하위 부서 리스트를 쿼리하여 조회
            java.util.List<String> subDeptList = loginService.getSubDeptList(param);
            userInfo.setSubDeptList(subDeptList);
        }
    }

    HttpSession httpSession = req.getSession();
    httpSession.setAttribute("ses", userInfo);
    UserInfoManager.createUserInfo(userInfo);
    utilService.logForJob("Login", "com.st_ones.common.login.service.LoginService", "Login", "Login", "Login", "I",  userInfo.getUserId(), req.getRemoteAddr(), "", userInfo.getUserType());
    messageService.setCommonMessage(httpSession);
}
```

### 4) 하위 부서 목록 조회 SQL 구현 ([LoginMapper.xml](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeCommon/src/main/resources/mappers/com/st_ones/common/login/LoginMapper.xml))
사용자의 부서 및 하위 조직코드 전체를 조회하는 계층형 쿼리 서비스를 추가합니다.

```xml
<select id="getSubDeptList" parameterType="hashMap" resultType="java.lang.String">
    <if test='_databaseId == "oracle"'>
        SELECT DEPT_CD 
          FROM STOCOGDP
         WHERE GATE_CD  = #{GATE_CD}
           AND BUYER_CD = #{BUYER_CD}
           AND DEL_FLAG = '0'
         START WITH DEPT_CD = #{DEPT_CD}
       CONNECT BY PARENT_DEPT_CD = PRIOR DEPT_CD
    </if>
    <if test='_databaseId == "mssql"'>
        WITH DEPT_CTE AS (
            SELECT DEPT_CD 
              FROM STOCOGDP
             WHERE GATE_CD  = #{GATE_CD}
               AND BUYER_CD = #{BUYER_CD}
               AND DEPT_CD  = #{DEPT_CD}
               AND DEL_FLAG = '0'

            SELECT Child.DEPT_CD
              FROM STOCOGDP Child
              INNER JOIN DEPT_CTE Parent ON Child.PARENT_DEPT_CD = Parent.DEPT_CD
             WHERE Child.GATE_CD  = #{GATE_CD}
               AND Child.BUYER_CD = #{BUYER_CD}
               AND Child.DEL_FLAG = '0'

        SELECT DEPT_CD FROM DEPT_CTE
    </if>
</select>
```

### 5) 구매요청 조회 XML 쿼리 반영 방안
사용자 세션의 `subDeptList`가 존재하는 경우(즉, 소속 회사의 권한 제어 설정이 켜져 있는 경우)에만 동적으로 부서 필터링을 수행하고, 그렇지 않으면 제한 없이 조회할 수 있도록 유연하게 적용합니다.

```xml
AND EXISTS (
    SELECT 1 
      FROM STOCUSER UR
     WHERE UR.GATE_CD  = PRHD.GATE_CD
       AND UR.USER_ID  = PRHD.REQ_USER_ID
       AND UR.DEL_FLAG = '0'
       <if test="ses.subDeptList != null and ses.subDeptList.size() > 0">
           AND UR.DEPT_CD IN 
           <foreach collection="ses.subDeptList" item="deptCd" open="(" close=")" separator=",">
               #{deptCd}
           </foreach>
       </if>
)
```

- **장점:** 
  - 로그인 시점에 1회 부서 계층 구조를 캐싱하므로 조회 화면 실행 시 성능 오버헤드가 없음.
  - 회사별로 권한 제어 여부를 간편하게 동적 제어(`deptSearchFlag`)할 수 있음.
