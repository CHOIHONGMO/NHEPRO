## 3. 검증 및 빌드 확인 사항

## 6. Maven systemPath 변수명 변경 (빌드 검증 로그 에러 제거)
* **작성 목적:** Maven 3.x 이상 빌드 시 `<systemPath>` 경로 내 `${basedir}` 변수 사용으로 인해 모델 검증 단계에서 발생하는 `[ERROR] ... must specify an absolute path` 경고/오류 로그를 제거하고 정석적인 절대 경로 변수 사용으로 개선
* **작성일:** 2026.06.15
* **변경 내용:**
  - **공통 변경 파일:**
    - [NHeCommon/pom.xml](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeCommon/pom.xml)
    - [NHeProFront/pom.xml](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProFront/pom.xml)
    - [NHeProOperator/pom.xml](file:///c:/ST-onesIDE/workspace/NHEPRO/NHeProOperator/pom.xml)
  - **세부 변경 내역:**
    - 각 모듈의 `pom.xml` 파일에 정의된 로컬 라이브러리(`system` 범위 의존성)들의 `<systemPath>` 경로를 기존 `${basedir}`에서 Maven 공식 권장 값인 **`${project.basedir}`**로 일괄 치환
      - 변경 전: `<systemPath>${basedir}/src/main/webapp/WEB-INF/lib/...`
      - 변경 후: `<systemPath>${project.basedir}/src/main/webapp/WEB-INF/lib/...`
  - **효과:**
    - Maven의 POM 모델 분석 및 빌드 수행 시 `[ERROR]` 수준의 불필요한 절대 경로 미지정 검증 경고 로그가 깔끔하게 제거됨을 확인
